# Sugar Rush 1000 — Math Engine Documentation

## Overview

Python-based math engine for the Sugar Rush 1000 slot game, built to match the **Stake Engine SDK** architecture. Simulates full game rounds — cluster evaluation, tumble cascades, multiplier progression, free spins — and outputs RGS-compliant files (`.jsonl.zst`, lookup table CSVs, `index.json`) for upload to the Stake Admin Control Panel (ACP).

**Tech Stack:** Python 3.11+ · zstandard (zstd compression)

---

## Architecture

The engine follows the Stake SDK's inheritance-based design. The `src/` directory provides generic, reusable base classes. The `games/sugar_rush_1000/` directory overrides them with Sugar Rush-specific rules.

```
                    ┌─────────────────┐
                    │  GeneralGameState│  (src/state/)
                    │  Board           │
                    └───────┬─────────┘
                            │ inherits
            ┌───────────────┼───────────────┐
            │               │               │
   ┌────────▼──────┐ ┌──────▼──────┐ ┌──────▼──────┐
   │  ClusterWins  │ │   Tumble    │ │  WinManager │
   │  (calculations)│ │(calculations)│ │   (wins)    │
   └────────┬──────┘ └──────┬──────┘ └─────────────┘
            │               │
            └───────┬───────┘
                    │ inherits
            ┌───────▼─────────┐
            │   Executables   │  (src/executables/)
            └───────┬─────────┘
                    │ inherits
     ┌──────────────┼──────────────┐
     │              │              │
┌────▼──────┐ ┌─────▼─────┐ ┌─────▼──────┐
│GameOverride│ │  Game     │ │  Game      │
│            │ │Executables│ │  State     │
└────────────┘ └───────────┘ └─────┬──────┘
                                   │ calls
                            ┌──────▼──────┐
                            │   run.py    │ → library/ output
                            └─────────────┘
```

## SDK Base Classes (`src/`)

### Config (`src/config/config.py`)

| Class | Purpose |
|-------|---------|
| `Config` | Base class. Reads reel CSVs, converts range-based paytables to flat lookup |
| `BetMode` | Defines a betting mode: name, cost multiplier, RTP target, max win, distributions |
| `Distribution` | One probability bucket: criteria name, quota (%), win criteria, conditions dict |

**Key Methods:**
- `convert_range_table(pay_group)` — Expands `((5,7), "H1"): 2.0` into `(5,"H1"):2.0, (6,"H1"):2.0, (7,"H1"):2.0`
- `read_reels_csv(filepath)` — Reads CSV, transposes rows→columns, returns `list[list[str]]`

### GeneralGameState (`src/state/state.py`)

The central state machine. Manages the game board, RNG seeding, win tracking, criteria checking, and simulation loops.

**Key Properties:**
| Property | Type | Purpose |
|----------|------|---------|
| `board` | `list[list[dict]]` | 7×7 grid (row-major). Each cell: `{symbol, id, explode, reel, row}` |
| `win_manager` | `WinManager` | Tracks all win amounts |
| `criteria` | `str` | Current distribution criteria (`"basegame"`, `"freegame"`, `"0"`, `"wincap"`) |
| `book` | `list[dict]` | Event log for current round (written to .jsonl.zst) |
| `wincap_triggered` | `bool` | True if 25,000× cap was reached |
| `repeat` | `bool` | True if current sim doesn't match criteria (must re-roll) |
| `_rng` | `Random` | Seeded RNG for reproducibility |

**Key Methods:**
| Method | Purpose |
|--------|---------|
| `reset_book()` | Clear board, book, wins for a new simulation |
| `reset_seed(sim)` | Seed RNG from simulation index |
| `check_repeat()` | Verify outcome matches distribution criteria (e.g., "wincap" must actually hit wincap) |
| `update_final_win()` | Calculate final win, enforce assertions |
| `run_sims(...)` | Main simulation loop: iterates N sims, assigns criteria, calls `run_spin()` |

### Board (`src/state/state.py`)

Inherits `GeneralGameState`. Implements reel-strip-based board generation.

**`create_board_reelstrips()`:**
1. Reads `reel_weights` from current distribution conditions
2. Selects reel strip ID (BR0, FR0, or SF0)
3. Picks a random stop position on each of 7 reels
4. Reads 7 consecutive symbols from each reel at that position
5. Transposes column-major → row-major for cluster evaluation
6. Populates `special_symbols_on_board` (scatters)

### ClusterWins (`src/calculations/cluster.py`)

**`get_cluster_data(record_wins=False)`:**
- BFS flood-fill on 7×7 grid with 4-directional adjacency (up/down/left/right)
- Skips scatter symbols (ID 7)
- Finds all connected groups of identical symbols
- Groups ≥ 5 cells = winning cluster
- Looks up payout from `config.paytable[(count, symbol_name)]`
- If `record_wins=True`, marks cells as `explode=True`
- Returns: `{"totalWin": float, "wins": [cluster_dict, ...]}`

### Tumble (`src/calculations/tumble.py`)

**`tumble_game_board()`:**
1. For each column: collect non-exploded cells, count exploded
2. Shift surviving cells to bottom (gravity)
3. Fill top positions from reel strip (moves backwards through strip)
4. Reassign row indices

### WinManager (`src/wins/win_manager.py`)

Tracks wins at three granularity levels:

| Level | Properties | When Updated |
|-------|------------|-------------|
| **Per cascade** | `spin_win`, `tumble_win` | `update_spinwin(amount)` |
| **Per round** | `basegame_wins`, `freegame_wins`, `running_bet_win` | `update_gametype_wins(type)` |
| **Cumulative** | `total_cumulative_wins`, `cumulative_base_wins`, `cumulative_free_wins` | `update_end_round_wins()` |

### Executables (`src/executables/executables.py`)

Ties everything together. Inherits from both `Tumble` and `ClusterWins`.

| Method | Purpose |
|--------|---------|
| `draw_board()` | Calls `create_board_reelstrips()` |
| `evaluate_wincap()` | Caps `running_bet_win` at 25,000×, trims overshoot |
| `count_special_symbols(key)` | Count scatters (or other specials) on current board |
| `check_fs_condition(key)` | Returns `True` if ≥3 scatters |
| `update_freespin_amount(key)` | Maps scatter count → free spin count via config |
| `run_freespin_from_base(key)` | Records scatter event, initializes FS, calls `run_freespin()` |

### WriteData (`src/write_data/write.py`)

Generates the three files required by Stake's ACP:

| Method | Output | Format |
|--------|--------|--------|
| `write_compressed_book()` | `books_<mode>.jsonl.zst` | zStandard compressed JSON-lines: `{id, events, payoutMultiplier}` |
| `write_lookup_table()` | `lookUpTable_<mode>_0.csv` | CSV: `ID, Weight(1), PayoutMultiplier(uint64)` |
| `write_index()` | `index.json` | `{"modes": [{name, cost, events, weights}]}` |

## Game Implementation (`games/sugar_rush_1000/`)

### GameConfig (`game_config.py`)

**Grid:** 7×7, cluster pays, min cluster size = 5

**Symbols (8 total):**

| ID | Code | Name | Frequency |
|----|------|------|-----------|
| 0 | L3 | Orange Gummy Bear | ~83% |
| 1 | L2 | Purple Gummy Bear | ~8% |
| 2 | L1 | Red Gummy Bear | ~5% |
| 3 | H4 | Green Candy | ~2% |
| 4 | H3 | Purple Candy | ~1% |
| 5 | H2 | Orange Candy | ~0.3% |
| 6 | H1 | Pink Candy (highest) | ~0.2% |
| 7 | S | Scatter | ~0.8% |

**Paytable (bet multipliers):**

| Cluster | L3 | L2 | L1 | H4 | H3 | H2 | H1 |
|---------|-----|-----|-----|-----|-----|------|------|
| 5 | 0.40 | 0.50 | 0.60 | 0.80 | 1.00 | 1.50 | 2.00 |
| 8 | 1.00 | 1.00 | 1.50 | 2.00 | 2.50 | 3.00 | 4.00 |
| 10 | 2.00 | 2.50 | 3.00 | 4.00 | 6.00 | 8.00 | 10.00 |
| 12 | 5.00 | 6.00 | 7.00 | 10.00 | 20.00 | 25.00 | 30.00 |
| 15+ | 40.00 | 50.00 | 60.00 | 80.00 | 120.00 | 200.00 | 300.00 |

**Scatter → Free Spins:** 3→10, 4→12, 5→15, 6→20, 7→30

**Win Cap:** 25,000× bet

**Multiplier Progression:** 0 → 1 (marked) → 2 → 4 → 8 → 16 → ... → 1024

**4 Bet Modes:**

| Mode | Cost | Description | Distributions |
|------|------|-------------|---------------|
| `base` | 1.0× | Normal play | 0.01% wincap, 4.9% FS, 45% zero-win, 50.09% basegame |
| `ante` | 1.25× | 2× scatter chance | 100% basegame with scatter override |
| `bonus` | 1000× | Buy Free Spins | 0.1% wincap, 99.9% FS |
| `super` | 500× | Buy Super FS (×2 start) | 100% FS with SF0 reel |

### GameExecutables (`game_executables.py`)

Sugar Rush-specific logic that extends the base `Executables`:

| Method | Purpose |
|--------|---------|
| `seed_super_free_spins()` | Set all 49 cells to ×2 multiplier |
| `advance_multiplier(r, c)` | `0→1→2→4→...→1024` at position (r,c) |
| `apply_cluster_multipliers(win_data)` | Sum all multipliers ≥2 on cluster positions, multiply base win |
| `evaluate_tumble_spin()` | Full cascade loop: cluster → multiply → record → advance → tumble → repeat |

**Cascade Loop Detail:**
```
1. Detect clusters (BFS)
2. Apply multipliers to each cluster's win
3. If no wins → break
4. Add win to WinManager
5. Log event to book
6. Check wincap (25,000×)
7. Advance multipliers on exploded positions
8. Tumble board (gravity + reel refill)
9. Go to step 1
```

### GameState (`gamestate.py`)

Combines `GameStateOverride` + `GameExecutables` via multiple inheritance.

**`run_spin(sim)`:**
```python
1. Seed RNG from simulation index
2. Loop (while self.repeat):
   a. Reset board, book, wins
   b. If super mode → seed ×2 multipliers
   c. Draw board from reel strips
   d. Run cascade loop (evaluate_tumble_spin)
   e. Update basegame wins
   f. If 3+ scatters and no wincap → trigger free spins
   g. Calculate final win
   h. Check if outcome matches distribution criteria
      - "wincap" criteria: must actually hit 25,000×
      - "0" criteria: must have zero wins
      - "freegame" criteria: must trigger free spins
      - If mismatch → set repeat=True → re-roll
3. Record cumulative wins
```

**`run_freespin()`:**
```python
1. Switch to freegame mode
2. For each free spin (multipliers persist across all spins):
   a. Draw new board
   b. Run cascade loop
   c. Check retrigger (3+ scatters → add more FS)
   d. Update freegame wins
3. End free spin round
```

### run.py — Simulation Entry Point

```python
create_books(gamestate, config, num_sim_args, ...)
```

1. For each bet mode (base, ante, bonus, super):
   a. Build `sim_to_criteria[]` from distribution quotas
   b. Run N simulations via `gamestate.run_sims()`
   c. Write `.jsonl.zst` compressed book
   d. Write lookup table CSV
2. Write `index.json`
3. Generate config files

## Reel Strips

Three CSV files, each containing 1000 rows × 7 columns:

| File | Used For | Characteristics |
|------|----------|-----------------|
| `BR0.csv` | Base game & Ante bet | Heavily weighted toward L3 (83%) |
| `FR0.csv` | Free spins | +40% H1, +30% H2, +50% scatter |
| `SF0.csv` | Super free spins | +20% H1, +20% scatter |

**Anti-clustering constraint:** No two adjacent symbols on any single reel are identical, which prevents guaranteed vertical clusters in a column.

**Symbol distribution on reel (after anti-clustering):**
```
L3: ~47%  |  L2: ~24%  |  L1: ~14%  |  H4: ~9%
H3: ~3%   |  H2: ~2%   |  H1: ~1%   |  S: ~0.8%
```

## RGS Output Format (library/)

After running `run.py`, the `library/` directory contains everything needed for ACP upload:

### index.json
```json
{
  "modes": [
    {"name": "base", "cost": 1.0, "events": "books_base.jsonl.zst", "weights": "lookUpTable_base_0.csv"},
    {"name": "ante", "cost": 1.25, "events": "books_ante.jsonl.zst", "weights": "lookUpTable_ante_0.csv"},
    {"name": "bonus", "cost": 1000.0, "events": "books_bonus.jsonl.zst", "weights": "lookUpTable_bonus_0.csv"},
    {"name": "super", "cost": 500.0, "events": "books_super.jsonl.zst", "weights": "lookUpTable_super_0.csv"}
  ]
}
```

### Lookup Table (CSV)
```
ID, Weight, PayoutMultiplier
0, 1, 210
1, 1, 700
2, 1, 2129
...
```
- Column 1: Simulation ID
- Column 2: Probability weight (1 = uniform, adjusted by optimizer)
- Column 3: Payout as uint64 (×100 of bet multiplier)

### Compressed Books (.jsonl.zst)
zStandard compressed JSON-lines. Each line:
```json
{"id": 42, "events": [{"index":0,"type":"cluster_win","wins":[...]}, {"index":1,"type":"cascade","grid":[[...]]}, ...], "payoutMultiplier": 2500}
```

## Tools

### generate_reels.py
Generates 1000-symbol reel strips with optimized symbol weights and anti-clustering.

```bash
python generate_reels.py
```

### optimize.py
Hill-climbing optimizer that searches for reel strip parameters producing 96.53% RTP.

```bash
python optimize.py
```
Runs three phases:
1. **Grid search** — Tests 35 weight/scatter combinations
2. **Hill climb** — Mutates weights toward target RTP
3. **Deep verification** — Multi-seed validation (10k+ spins)

### rtp_test.py
Runs N spins through the full engine and reports RTP, max win, and free spin hit rate per mode.

```bash
python rtp_test.py
```

## Quick Start

```bash
# 1. Install dependency
pip install zstandard

# 2. Generate reel strips (already included, but can regenerate)
python generate_reels.py

# 3. Run optimizer to tune RTP (optional — takes minutes to hours)
python optimize.py

# 4. Generate RGS output files
python games/sugar_rush_1000/run.py

# 5. Verify RTP
python rtp_test.py

# Output is in: games/sugar_rush_1000/library/
```
